# 規劃：新增 License 頁面

## 目標 (Goal)
在 SettingsPage 的末端新增一個 License ListTile，點擊後觸發 Flutter 內建的 `showLicensePage()`，以顯示 App 的名稱與版本號。

## 現有架構分析
目前的設定頁面實作於 `lib/settings/settings_screen.dart`，主要使用 `ListView` 結合自訂的 `_SettingsLine` 組件來呈現各個設定項目。語言包位於 `assets/translations/`，目前支援 `en`、`zh`、`ja` 三種語系。

專案中並未安裝 `package_info_plus`，因此可以直接在 `showLicensePage()` 傳入固定的 `applicationName` 與 `applicationVersion`（參考 `pubspec.yaml`：版本為 `0.0.1+1`）。

## 實作步驟

### 1. 更新多語系字串
在所有語系的 JSON 檔案中新增對應的 key：
- `assets/translations/en.json`: 新增 `"license": "License"`
- `assets/translations/zh.json`: 新增 `"license": "授權資訊"`
- `assets/translations/ja.json`: 新增 `"license": "ライセンス"`

### 2. 在 SettingsScreen 新增 UI 元件
在 `lib/settings/settings_screen.dart` 中：
- 於 `ListView` 的 `// QR Code Display` 區塊上方（或重置進度按鈕下方），新增一行 License 選項。
- 使用現有的 `_SettingsLine`，傳入：
  - 標題：`'license'.tr()`
  - 圖示：`const Icon(Icons.info_outline)` 或是 `const Icon(Icons.description)`
  - `onSelected` 事件。

### 3. 實作 `showLicensePage` 呼叫機制
在 License 選項的 `onSelected` 回呼函式中，呼叫：
```dart
showLicensePage(
  context: context,
  applicationName: 'OK Multipl Poker',
  applicationVersion: '0.0.1+1', // 對應 pubspec.yaml 的版號
  applicationIcon: const Image(
    image: AssetImage('assets/icons/app_icon_card.png'),
    width: 64,
    height: 64,
  ),
);
```
這會利用 Flutter 內建功能自動列出應用程式所需的所有第三方程式庫授權（透過 LICENSE 檔案及 pubspec 解析）。

## 邊界條件與測試基準
- 點擊設定頁面上的 License 按鈕能成功開啟 Flutter 預設的 License 頁面。
- 顯示的 App 標題和版本號要與 `pubspec.yaml` 內保持一致。
- 確保 License 標題正確支援國際化。
